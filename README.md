# bakery-reservation# 🍞 인기 베이커리 픽업 예약 시스템

> 한정된 인기 디저트를 픽업하기 위한 실시간 재고 조회 및 예약 서비스

## 무엇을 하는 프로젝트인가
오프라인 매장에 손님이 몰려 발생하는 긴 대기열과 헛걸음을 방지하기 위해 기획된 웹 서비스입니다. 사용자는 남은 빵의 수량을 실시간으로 확인하고, 혼잡하지 않은 빈 시간대를 골라 미리 결제하여 QR코드로 픽업할 수 있습니다. 

## 사용 방법
1. 브라우저에서 제공된 링크(GitHub Pages)에 접속합니다.
2. 아무 아이디나 입력하여 테스트 로그인 후 기능을 사용해 볼 수 있습니다.
3. 원하는 상품과 수량을 선택하고 장바구니에 담습니다.

## 🚨 아직 안 되는 것 (부숴보며 찾은 한계점)
AI 에이전트를 통해 코드를 생성하고 '내 앱 망가뜨리기 게임'으로 직접 테스트해 본 결과, 다음과 같은 심각한 버그와 한계점을 발견했습니다.

1. **마이너스 수량 입력 버그 (어뷰징 취약):** 수량에 `-5` 같은 음수를 입력하고 담기를 누르면, 차감되어야 할 재고가 오히려 플러스로 늘어나는(예: 23/20) 치명적인 오류가 발생합니다.
2. **연쇄적인 UI 마비:** 위처럼 비정상적인 값을 입력하고 나면, 다른 빵들의 '담기' 버튼이 전부 회색으로 비활성화되어 더 이상 정상적인 이용이 불가능해집니다.
3. **시간대 렌더링 오류:** 상품을 정상적으로 담아도 '픽업 시간대 선택' UI가 나타나지 않거나 제대로 동작하지 않아, 다음 단계인 '결제 및 QR 발급'으로 넘어갈 수 없습니다.
4. **프론트엔드 한계:** 데이터베이스가 연결되어 있지 않아 새로고침 시 모든 데이터가 날아갑니다.
